Description

The aim of this project is to create an end-to-end big data pipeline which ingests the large data from a relational database into HDFS followed by analyzing the distributed data (batch processing) using Spark Engine and presenting the results in a dashboard. The stages of the pipeline are as follows:

  1. Data Collection and Ingestion
    • Building a relational database using PostgreSQL and importing relational data to HDFS using Sqoop and Hadoop MapReduce engine.
  2. Data Storage/Preparation
    • Creating Hive Tables with (AVRO) serialized and (Snappy) compressed file formats.
      • Storing the data in the data warehouse Hive for further analytics.
  3. Data Analysis
    • Perform Exploratory Data Analysis (EDA) with HiveQL on Tez engine and SparkSQL on Spark engine.
    • Perform Predictive Data Analysis (PDA) via building ML models using Spark MLlib.
      • This part also includes feature extraction and data preprocessing.
  4. Presentation
    • Presenting the analysis results in a dashboard using Apache Superset.

The deliverables of this project are:

  1. The project repository
    • You have to use only Python programming language in addition to SQL and HiveQL for EDA and data preparation.
    • You have to use and follow the project repository template attached to this tutorial for your project. Do not fork but duplicate it and start working on your project.
      • The repository has a README file which includes the rules of using the template.
      • The scripts should generate reproducible results. For instance, If I run the script stage1.sh for the second time, it should not give me any errors, so you should make sure that you cleared the objects before creating new ones.
    • For EDA part
      • You need to analyze the data and display data characteristics and at least 5 different data insights in the dashboard prior to data preprocessing.
      • Data insights should be valuable and help business stakeholders make better decisions
      • For each insight drawn from the data, you need to create charts and add a description for it.
        • You need to explain the data insight (data storytelling).
          • Data storytelling is the ability to effectively communicate insights from a dataset using narratives and visualizations. It can be used to put data insights into context for and inspire action from your audience.
    • For ML part
      • You need to build two different kinds of models like Random Forest and Naive Bayes for classification task.
      • For data split, use 70% training and 30% test. You could use different ratios based on the size of your dataset but you should always make sure that you have at least 60% training data.
      • Fine-tune the hyper-parameters via grid search on 2 different values for each hyperparameter. It is enough tp optimize for only 4 different combinations. The number of epochs/iterations is not considered as a hyperparameter.
      • You should use the following performance metrics (reference):
        • Accuracy and F1 for multiclass classification tasks
          • Area Under ROC and Area Under PR for binary classification tasks.
        • RMSE and R2R2 for regression tasks.
        • Mean Average Precision and NDCG for recommendation tasks.
      • Perform k-fold cross validation (where 1<k<5) and select the best model.
      • Show the performance of both best models of each kind in the dashboard.
    • For the dashboard.
      • It is enough to create a single dashboard using Apache Superset.
      • You have to display the data characteristics such as number of data instances, number of features, feature names,…
      • You need to display all the results of the analytics.
      • You need to train the model on a cluster (Hadoop YARN) and not local machine.
      • Check the grading criteria to see the requirements needed to be fulfilled here.
  1. A summary report

Dataset Criteria

  1. Data is big in terms of volume for batch processing on Spark engine.
    • At least 20,000 data samples.
  2. At least 7 explanatory variables (features). The features should not be anonymized nor encoded. The features should be described.
  3. Either datetime or geospatial features must be present in the data (both can be).
  4. The goal of predictive analysis with regard to the data should be detemined (Regression, Classification, …etc).
  5. The dataset should not be one of the datasets you worked on during the labs or assignments.
  6. Preferred format of the dataset files is “csv”. JSON and Excel files are fine but you need to convert them to csv.
  7. The TA needs to verify your dataset
  8. You should find a dataset before March 31.

Team Formation Criteria

  1. Each teammate should participate in the project and presentation.
  2. You have to work in teams of 3-4 people.
    • For exceptional situations, you need to contact the TA.
  3. You should select a team before March 31.

Grading Criteria

  • [10 points] Stage I
    • [5 points] You built all tables in the database. You added all constraints if any. You loaded the data to the tables.
    • [5 points] You imported the data to HDFS via Sqoop.
  • [20 points] Stage II
    • [5 points] You created Hive Tables and stored the data.
    • [10 points] EDA
      • [2 points] for each insight provided
  • [25 points] Stage III
    • [25 points] PDA
      • [9 points] for the first model
      • [9 points] for the second model
      • [5 points] for cross validation and hyperparameter tuning
      • [2 points] for prediction of a specific data sample
  • [10 points] Stage IV
    • [10 points] for the dashboard
      • [2 points] for showing the data characteristics
      • [3 points] for the insights with provided description
      • [2 point] for showing the performance of models
      • [2 points] for prediction results
      • [1 points] for visual quality of dashboard
  • [15 points] Report
  • [15 points] Project Defence
    • [5 points] for presentation
    • [10 points] for answering the questions
  • [5 points] Check code quality using pylint
    • [4 points] if only 2-3 warnings raised
    • [3 points] if only 4 warnings raised
    • [2 points] if only 5-6 warnings raised
    • [1 points] if 7+ warnings raised
