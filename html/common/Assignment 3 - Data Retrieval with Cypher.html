Description

This assignment will be dedicated to practice on building a Neo4j graph database and writing Cypher queries.

You have to work on check-in dataset collected from Foursquare, which has been widely used as a key data source for studying location based services. Specifically, as Foursquare user’s check-ins can only be accessed from the user’s social circle, they are not publicly available. However, many Foursquare users choose to also share their check-ins or points of interest (POIs) via Twitter. User social relationships are collected from Twitter. A friendship exists between two users if the two users follow each other. The user data is anonymized for privacy purposes.

The dataset includes long-term (about 22 months, from April 2012 to January 2014) global-scale Foursquare check-in data, as well as two snapshots of user social networks before (in Mar. 2012) and after (in May 2014) the check-in data collecting period. The check-in dataset includes 22,809,624 checkins from 114,324 users across 3,820,891 venues. There are 363,704 (old) and 607,333 (new) friendships in the social network data.

Data Description

The dataset consists of 5 tsv files without headers. We present in the table below the description of the dataset files. Use it to know the headers of the files.

File Name Description Fields
checkins_anonymized.txt Each line corresponds to a user id checked in a venue at time UTCTime with Timezone offset in minutes user_id, venue_id, utc_time, timezone_offset_mins
friendship_before.txt Each line corresponds to a user id at the first column mapped to the friend id at the second column and they have friendship before April 2012 user_id, friend_id
friendship_after.txt Each line corresponds to a user id at the first column mapped to the friend id at the second column and they still have friendship after January 2014 user_id, friend_id
POIs.txt Each line corresponds to the features of the venue including venue id, POI latitude, POI longitude, venue category name, country code (ISO 3166-1 alpha-2 two-letter country codes) venue_id, latitude, longitude, category, country
users.txt a file with a header where each line contains user id for all 114,324 users userid

Instructions

In this assignment, you need to extract your slice of data using pandas on Zeppelin or Jupyter or any data analysis tool with the corresponding users’ checkins and friendships. Add commands and screenshots of output to the report in task 1.

After that, you need to build your graph database only for the slice fo data you extracted based on your sid. After building the database (loading all data), you need to get the database schema by running the following command in Neo4j Browser.

call db.schema.visualization()

You should run this command after building the database or before you submit to Moodle.

Take the screenshot for the schema and submit it to the report for the task 2. For each next task, you need to analyze the data and put the query as a text and a screenshot of the output. Please do not change the order of the queries.

You need to submit only the report week3.pdf. The total points for this assignment is 100.

Note: Some queries do not return graphs, so the text representation of the output need to be added as a screenshot to the report. If the output is long and does not fit into one screenshot then add the single screenshot and the text representation of the output.

Exercises

The Neo4j database foursquare in this assignment has the following schema.

Schema of our Neo4j database

As you can see, there are two node labels User which has the property userid and Venue which has the properties {venue_id, latitude, longitude, category, country}. The relationships are CHECK_IN which has the properties {utc_time, timezone}, FRIEND_BEFORE and FRIEND_AFTER. The frienships need to be in both directions since it is a mutual relationship.

  1. [10 points] Extract your slice of users data and their corresponding check-ins and friends. Given the whole dataset files from the attached link, you need to prepare your data files. Specifically, you need to do as follows:
    • Prepare the friendships_before of your users with their friends and store it in “my_frienship_before.tsv”.
    • Prepare the friendships_after of your users with their friends and store it in “my_frienship_after.tsv”.
    • Prepare the check ins of your users and friends and store it in “my_checkins_anonymized.tsv”.
    • Prepare the POIs of your users and friends and store it in “my_POIs.tsv”.
    • Throw users.csv and all other downloaded files to Recycle Bin (better to remove them completetly ).
    • Add to the report the commands and screenshots of the output of the commands.
  2. [10 points] Build the Neo4j database foursquare according to the given schema. Load all data to the database. Add all commands of building the database to the report. Add the screenshot of the schema of your database to the report.
  3. [4 points] What is the most common country in your slice of data?
  4. [4 points] Who are the users who shared check-ins in 2013?
  5. [4 points] Who are the friends of the first 10 users?
  6. [6 points] What are the categories of venues in each country? Retrieve only the first 10 results. The query should retrieve countries and category of venues.
  7. [6 point] Which are the user ids, venue ids, venu categories, local datetime of the check-ins in the most common country in your slice of data? Notice that local datetime of the check-ins should be calculated from the given offset and utc time.
  8. [6 point] What are the friend ids of users who checked in the most common country in your slice of data and have friendship before April 2012?
  9. [6 point] For each user, What is the number of friends before April 2012 and number of friends after Januray 2014? also return the change in number of friends.
  10. [8 point] Retrieve the categories of venus which are checked in by friends of first 10 users from your slice and have friendship before April 2012.
  11. [8 points] Retrieve the top 5 recent venues which are checked in by friends of users who also checked in these top 5 recent venues.
  12. [6 points] Retrieve the average number of users checked in for each venue category.
  13. [8 points] Retrieve first 10 friends of the users who checked in the most common category of venues in your slice of data
  14. [4 points] Retrieve first 10 friends of friends of the users who checked in the most common category of venues in your slice of data
  15. [10 points] Retrieve first 10 friends of friends of friends of the users who checked in the most common venue in your slice of data and all have friendship with each other before April 2012 or after January 2014.
