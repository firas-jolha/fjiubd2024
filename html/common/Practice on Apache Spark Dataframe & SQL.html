Description

This practice will be dedicated to analyze the data via Spark DataFrame and Spark SQL APIs. You have to work on daily statistics for trending YouTube videos collected from Youtube via an open source tool called Trending-YouTube-Scraper. YouTube (the world-famous video sharing website) maintains a list of the top trending videos on the platform. According to Variety magazine, “To determine the year’s top-trending videos, YouTube uses a combination of factors including measuring users interactions (number of views, shares, comments and likes). Note that they are not the most-viewed videos overall for the calendar year”. Top performers on the YouTube trending list are music videos (such as the famously virile “Gangam Style”), celebrity and/or reality TV performances, and the random dude-with-a-camera viral videos that YouTube is well-known for. This dataset is a daily record of the top trending YouTube videos.

This dataset includes several months (and counting) of data on daily trending YouTube videos. Data is included for the US, GB, DE, CA, FR, RU, MX, KR, JP and IN regions (USA, Great Britain, Germany, Canada, France, Russia, Mexico, South Korea, Japan and India respectively), with up to 200 listed trending videos per day.

Each region’s data is in a separate file. Data includes the video title, channel title, publish time, tags, views, likes and dislikes, description, and comment count.

The data also includes a category_id field, which varies between regions. To retrieve the categories for a specific video, find it in the associated JSON. One such file is included for each of the 10 regions in the dataset

Note: In this practice, it is enough to retrive and analyze data relevant to a single region. You can select a specific region and analye its relevant data.

Dataset Description

The dataset for each region consists of a csv data file and json file for category titles. The data file consists of the following fields:

  • video_id (Common id field to video csv file)
  • trending_date (trending date of the video)
  • title (video title)
  • channel_title (channel title for the video)
  • category_id (Can be looked up using the included JSON files, but varies per region so use the appropriate JSON file for the CSV file’s country)
  • publish_datetime (publish datetime for the video)
  • tags (Separated by | character, [none] is displayed if there are no tags)
  • views (number of views for the video)
  • likes (number of likes for the video)
  • dislikes (number of dislikes for the video)
  • comment_count (number of comments for the video)
  • thumbnail_link (number of comments for the video)
  • comments_disabled (disabled comments for a video means that the video does not have comments)
  • ratings_disabled (disabled ratings for a video means that the video does not have likes or dislikes)
  • video_error_or_removed (status of the scrpaing the video metadata)
  • description (video descrption)

The JSON file consists of a root object which includes a field items for an array of category info. The category id is located in id field of the array element and the category title is located in snippet.title.

In this practice, you need to analyze the data on Spark engine where the queries can be written on the pyspark shell or in a Zeppelin note with python intrepreter or pyspark interpreterl.

You can write the queries in a file file.py and submit it to Spark as follows:

spark-submit <file.py>

Exercises on Mongodb

  1. Explore the provided JSON file using any library like pandas (We can use pandas here since we consider that this file is not big to analye it using Spark) and extract category titles, and store them as a dictionary of key value pairs where the key is the category id and the value is the category title. Store this dictionary as a new collection categories in a new db youtube in your Mongodb server (use pymongo package).

Exercises on Spark DataFrame

Note: In this list of exercises, you have to use the pyspark.sql.DataFrame API. You need to perform the analysis using Spark DataFrame API.
2. Read the collection categories from Mongodb database youtube and store it in a Spark Dataframe.
3. Select a region and get its csv file.
4. Build a schema for the dataset in csv file and use the schema to read the data.
5. Retrieve the distinct video titles sorted in alphabetic order.
6. Retrieve the number of times/days that a video became trend based on the available dataset. Show the video id, title and number of times it was trend.
7. Retrieve the video ids which were trend more than one time/day and save it in HDFS /data/trends. Check the output folder and see how many files you got and the content of the files. Repartition the dataframe into 4 partitions and save it in HDFS again for further analysis. Note: If you do not have HDFS, then store it in the local disk for simplicity.
8. Read the folder data/trends into a dataframe trend_videos. For each trend video from trend_videos, select their most recent stat info and store it in another dataframe trend_video_stats.
9. Create a dataframe contains the id of the video, its title, its publish time, its category id, the title of the category. Show only first 10 recently published videos. For videos which were trend more than one time/day, you need to use the dataframe trend_videos.
10. Show the videos’ most recent stat info which have more than 1 million views. Sort the results based on number of views and recent trend date.
11. Count the number of videos per channel and retrieve the top 5 channels.
12. What are the top-5 videos titles based on number of reviews for the top 5 channels?
13. Calculate the difference in likes and dislikes for videos published before 2010 and show the top-10 videos in descending order.
14. Calculate the average likes for each channel and display the top 5 channels.
15. Calculate the total views for each channel (sum of views per each video with the most recent stat info) and retrieve the channels which have more than 1 million views in total.
16. Analyse the linear correlation between average number of views and likes for all videos (with the most recent stat info) and explain whether the correlation is significant or not. Use 5% significance level.
17. Calculate the average number of tags for each channel. Return the top-10 channels which have most tags.
18. Calculate the frequency of tags in all videos and display the top-10 tags in all videos with its frequency (number of videos).
19. Calculate the max number of comments in each channel and display the top 10 channels with its max comment count.
20. Define a UDF for extracting the links which start with https: or http: and the mentions which start with @.
21. Search in the description of videos for the links and mentiones and display the links and mentiones for each video as a list. Sort the result according to the video title in ascending order.
22. Find the videos which were trend in two consecutive days.

Exercises on Spark SQL

Note: In this list of exercises, you have to use the spark.sql function. You need to perform the analysis using Spark SQL queries.
23. Calculate the difference in likes and dislikes for each video and show the top-10 videos in descending order.
24. Calculate the average likes for each channel and display the top 5 channels.
25. Calculate the total views for each channel and retrieve the channels which have more than 1 million views in total.
26. Calculate the average number of tags for each channel. Return the top-10 channels.
27. Calculate the frequency of tags in all videos and display the top-10 tags in all videos with its frequency (number of videos).
28. Calculate the max number of comments in each channel and display the top 10 channels with its max comment count.
